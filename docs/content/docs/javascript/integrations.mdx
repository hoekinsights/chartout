Pick an embedding path based on your app stack. All paths share the same model and store helpers; only the mount step differs.

| Your app | Import | You provide |
|----------|--------|-------------|
| [React](#react) | `chartout/react` | `react`, `react-dom` |
| [Vanilla JS / static HTML](#vanilla-js) | `chartout/mount` | nothing extra |
| [Vue](#vue) | `chartout/mount` | nothing extra |
| [Svelte](#svelte) | `chartout/mount` | nothing extra |

## Package exports

| Export | Purpose |
|--------|---------|
| `chartout` | `createChartoutModel`, state types, `Theme`, `themeToStyle` |
| `chartout/store` | `openWithViz`, `openWithItem`, `openWithCart`, `svgToBytes`, `products` |
| `chartout/react` | `ChartoutWidget` for React apps |
| `chartout/mount` | `mount()` for vanilla JS, Vue, Svelte, and other non-React hosts |

`chartout/react` is for React 19+ apps: install `react` and `react-dom` as peers, and the widget shares your React instance. `chartout/mount` is self-contained for other hosts.

```bash
npm install chartout react react-dom
```

  <Tab value="React" id="react">
    Use when your app already runs React 19 or newer. The widget shares your React instance (no duplicate copy).

    ```bash
    npm install chartout react react-dom
    ```

    ```tsx
    import { createChartoutModel } from 'chartout'
    import { ChartoutWidget } from 'chartout/react'
    import { openWithViz } from 'chartout/store'

    const model = createChartoutModel({})

    function App() {
      return (
        <>
          <button onClick={() => openWithViz(model, svg, 'My chart')}>
            Print this chart
          </button>
          <ChartoutWidget
            model={model}
            theme={{ accent: '#db2777', radius: '12px' }}
            style={{ width: '100%' }}
          />
        </>
      )
    }
    ```

    **Gotchas**

    - Install `react` and `react-dom` as peers; `chartout/react` does not bundle them.
    - Set width on the container (e.g. `width: '100%'`). Height is fixed; see [Layout](#layout).
    - Call `openWithViz`, `openWithItem`, or `openWithCart` after the widget is mounted (or anytime; state syncs through the model).
  </Tab>
  <Tab value="Vanilla JS" id="vanilla-js">
    Use for plain HTML, Electron renderers, or any page that can load an ES module.

    ```html
    <div id="store" style="width:100%"></div>
    <button id="print-btn">Print this chart</button>
    ```

    ```ts
    import { createChartoutModel } from 'chartout'
    import { mount } from 'chartout/mount'
    import { openWithViz } from 'chartout/store'

    const model = createChartoutModel({})
    const { destroy } = mount(document.getElementById('store')!, {
      model,
      theme: { accent: '#db2777' },
    })

    document.getElementById('print-btn')!.onclick = () =>
      openWithViz(model, svgElement, 'My chart')

    // Optional cleanup when navigating away:
    // destroy()
    ```

    **Gotchas**

    - Call `destroy()` when removing the container from the page.
    - Give the container a width before mounting. Height is fixed; see [Layout](#layout).
  </Tab>
  <Tab value="Vue" id="vue">
    Use `chartout/mount` with a container ref. Mount when the component appears; call `destroy()` when it is removed.

    ```vue
    <script setup lang="ts">
    import { onMounted, onUnmounted, ref } from 'vue'
    import { createChartoutModel } from 'chartout'
    import { mount } from 'chartout/mount'
    import { openWithViz } from 'chartout/store'

    const container = ref<HTMLElement | null>(null)
    let destroy = () => {}

    onMounted(() => {
      if (!container.value) return
      const model = createChartoutModel({})
      ;({ destroy } = mount(container.value, { model }))
      // store model on a ref if you need openWith* from other handlers
    })

    onUnmounted(() => destroy())
    </script>

    <template>
      <div ref="container" style="width:100%" />
    </template>
    ```

    **Gotchas**

    - Keep one model instance per widget; create it in `onMounted` or hoisted to module scope if the component never unmounts.
    - Always call `destroy()` on unmount to release the widget.
  </Tab>
  <Tab value="Svelte" id="svelte">
    Use `chartout/mount` with a bound container element. Mount in `onMount`; call `destroy()` in `onDestroy`.

    ```svelte
    <script lang="ts">
    import { onMount, onDestroy } from 'svelte'
    import { createChartoutModel } from 'chartout'
    import { mount } from 'chartout/mount'
    import { openWithViz } from 'chartout/store'

    let container: HTMLDivElement
    let destroy = () => {}

    onMount(() => {
      const model = createChartoutModel({})
      ;({ destroy } = mount(container, { model }))
    })

    onDestroy(() => destroy())
    </script>

    <div bind:this={container} style="width:100%"></div>
    ```

    **Gotchas**

    - Same lifecycle rules as Vue: one `destroy()` per mount.
    - For SvelteKit, import `chartout/mount` only in client-side code (`+page.svelte` or `onMount`), not in universal server loads.
  </Tab>

## Widget bundle

The store is a **browser-only** UI. It needs a DOM and does not run on a server.

`chartout/mount` and `chartout/react` ship a prebuilt widget bundle. You do not install its internals separately. The bundle already includes:

- **Three.js** for the 3D product preview
- **skia-canvas** for compositing chart PNGs onto the product mockup
- **anywidget** utilities and other UI code

You do not add these to your `package.json`. If you use a meta-framework with server-side rendering, a server build may fail on `skia-canvas` even though the widget only runs in the browser. Import chartout in client code only; see [SSR and meta-frameworks](#ssr-and-meta-frameworks) if the build still errors.

### Layout

The store UI has a fixed height of about 357px, sized for notebook cell output where chartout was first embedded. The layout is fully responsive in width; set the container width (e.g. `width: '100%'` on the mount container or `style` prop on `ChartoutWidget`).

Flexible height may be supported in a future release. [Open an issue](https://github.com/hoekinsights/chartout/issues) if that matters for your app.

### svgToBytes

`svgToBytes` in `chartout/store` rasterises an SVG chart to PNG bytes using the browser's `OffscreenCanvas` API. That produces the print content for cart placements.

Call it yourself when building `CartItem[]` for `openWithCart`. For `openWithViz` and `openWithItem`, pass an SVG and those helpers call `svgToBytes` internally.

It lives in the slim `chartout/store` bundle, separate from the widget UI.

### SSR and meta-frameworks

If a server build pulls in `chartout/mount` or `chartout/react`, it may resolve `skia-canvas` incorrectly and fail with errors about `.node` files.

Import chartout only in client code (`'use client'` in Next.js, `onMount` in Svelte, client-only components in Nuxt). If the server build still mentions `skia-canvas`, exclude it from the server bundle:

```ts
// vite.config.ts
export default defineConfig({
  ssr: {
    external: ['skia-canvas'],
  },
})
```

```ts
// next.config.ts
webpack: (config, { isServer }) => {
  if (isServer) {
    config.externals = [...(Array.isArray(config.externals) ? config.externals : [config.externals].filter(Boolean)), 'skia-canvas']
  }
  return config
},
```

## Browser APIs

| API | Used by |
|-----|---------|
| `OffscreenCanvas` | `svgToBytes` |
| `fetch` | `products()`, widget catalogue load |
| WebGL | 3D product preview |

Target modern evergreen browsers. Ensure chart SVGs have readable `width`/`height` or a `viewBox` before calling `svgToBytes`.
